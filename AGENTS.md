## Project Execution Rules (Minimal)

이 프로젝트는 작업 진행 일관성을 위해 아래 규칙을 기본으로 적용한다.

1) TODO 표시 및 단계 수행
- 모든 진행 요청 시작 시 채팅 UI에 TODO 체크리스트를 먼저 표시한다.
- 체크리스트 상태는 `pending`, `in_progress`, `completed`로 갱신한다.
- 동시에 `in_progress`는 1개만 유지한다.
- 권장 표기:
  - `[ ]` pending
  - `[~]` in_progress
  - `[x]` completed

2) 완료 전 검증 필수
- 구현 완료 선언 전 반드시 `pnpm build`를 실행해 성공을 확인한다.
- `pnpm build`가 미완료 또는 실패 상태면 완료로 보고하지 않는다.

3) 테스트 정책
- 테스트 실행 여부는 사용자 최신 지시를 우선한다.
- 현재처럼 UI 조정 단계에서 테스트 제외 지시가 있으면 이를 따른다.

4) 범위 원칙
- 앱 도메인 API/타입 변경이 없는 문서성 작업은 문서만 수정한다.
- 불필요한 구조 변경은 하지 않는다.
